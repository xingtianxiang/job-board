# 团队看板 · 使用指南(给同事)

这是我们项目的"对齐看板":一眼看清**有哪些模块、各自边界、谁在做什么、统一的技术方案、功能进度**。
目的是大家(尤其是用 AI 编程时)不越界、不撞车、技术路线一致。

> **看之前先填两处**(项目负责人填好再发,或单独告诉你):
> - 网站地址:`【网址】`(例如 https://job-board-xxx.vercel.app)
> - 上传令牌:`【上传令牌】`(只有需要本地更新看板的人才用到,见第三部分)

---

## 一、登录(1 分钟)

1. 打开 **`【网址】`**。
2. 第一次会进到"选名字"页:
   - 名单里**有你的名字** → 直接点它进入。
   - **没有** → 点 **"+ 新增成员"** → 输入名字、选一个高亮色 → "创建并进入"。
3. 之后浏览器会记住你,下次直接进看板,不用再选。

> 🔑 **重要提示**:你的名字最好和你电脑上的 git 用户名**一致**。
> 在终端运行 `git config user.name` 看一下,新增成员时就用这个名字。
> (这样你改了代码后,对应模块会自动高亮成**你的颜色**;名字对不上会显示成默认橙色。)

---

## 二、怎么看这张看板

**中间的模块地图**(最重要):
- 每个**方块 = 一个模块/子系统**;方块之间的箭头 = 依赖关系。
- **方块颜色 = 正在做这块的人的颜色**:
  - `✎ 某人 在改` = 那个人本地正在改这个模块的代码(自动检测)。
  - `▶ 进行中` = 这块有标记为"进行中"的功能。
  - **白色** = 当前没人在动。
- 方块下面的 **"负责:XXX"** = 这个模块的负责人。
- **点一个方块** → 右侧弹出抽屉:看这个模块的**边界(负责什么/不负责什么)**、进行中的功能、相关技术决策。
- 抽屉里点 **"进入模块单页"** → 看这个模块的完整文档。

**顶部"谁在做什么"**:自动汇总大家当前在做的事。

**右侧"统一技术方案 / 决策"**:团队已定的技术决策(为什么这么做)+ 顶部钉住的"当前约定技术栈"。**动手前先扫一眼这里,别和已定方案打架。**

**底部"产品功能与状态"**:待办 / 进行中 / 完成 三列。每张卡左下小圆点 = 负责人色。完成太多可以点 **"全部归档"** 收起,到右上角 **"已归档 →"** 里随时找回。

---

## 三、配置:认领你负责的模块

进看板后,把你负责的模块标上你:
1. 点你负责的模块方块 → 抽屉里 **"负责人"** 下拉 → 选你自己。
2. 这块以及它名下的功能卡,就会标成你的色。

这步**人人都该做一下**,这样大家一眼知道谁管哪块、边界在哪。

## 四、(进阶 · 可选)让你的代码改动自动上看板

如果你也要**更新看板**(让你正在改的模块自动高亮、或更新进度),需要在本地装一次同步工具。**只想看的人可以跳过这部分。**

**一次性准备:**
1. 装 **Node 20+**(已有更高版本也行)。没有就去 nodejs.org 装 LTS,或用 nvm。
2. 拿到看板工具代码并装依赖:
   ```bash
   git clone https://github.com/xingtianxiang/job-board.git
   cd job-board
   npm install
   ```
3. 在 `job-board` 目录建一个 `.env` 文件,写一行(令牌找项目负责人要):
   ```
   BOARD_INGEST_TOKEN=【上传令牌】
   ```

**日常更新(在 job-board 目录里跑):**
- 你只要**正常写代码**。想把"我在改哪块"推上看板时:
  - PowerShell:
    ```powershell
    $env:BOARD_URL="【网址】"
    npm run sync -- 你的项目路径\BOARD.md
    ```
  - 例如项目和 job-board 同级时:`npm run sync -- ..\weld_KAIERDA\BOARD.md`
- 它会自动看你**当前(未提交)改了哪些文件** → 对应模块高亮成你的色;并把 `BOARD.md` 里的进度同步上去。

**更新看板内容(模块/进度/决策)**:改项目根目录的 `BOARD.md` 即可。可以直接让你的 AI(Claude / Codex 都行)帮你改——对它说:
> "按 BOARD_PROMPT.md 增量更新 BOARD.md"

(`BOARD_PROMPT.md` 在 job-board 目录里,是一份工具无关的指令模板。)改完再 `npm run sync` 一下就生效。

## 五、常见问题

- **我的高亮是橙色,不是我选的颜色?** 你的成员名字和 git 用户名(`git config user.name`)对不上。把成员名改成和 git 一致(可在登录页删掉旧的、重新"新增成员")。
- **整张图都被点亮了?** 一般是你在一个改了很多文件的大分支上;聚焦到具体改动(未提交的)时就只亮那几个。
- **完成项太多?** 点"全部归档"收起,"已归档 →"里能找回。
- **改了看板没变?** 内容更新要在本地 `npm run sync` 一下;网站功能更新由负责人部署后**强刷(Ctrl+F5)**即可。
- **登录要密码吗?** 不要,内部工具,选名字即可。

有问题直接找项目负责人。
